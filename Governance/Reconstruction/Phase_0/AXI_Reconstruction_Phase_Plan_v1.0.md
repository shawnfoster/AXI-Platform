# AXI Reconstruction Phase Plan v1.0

**Document ID:** AXI-RP-001  
**Version:** 1.0  
**Status:** Founder Review  
**Lifecycle:** Draft

## Purpose
Define the governed sequence of phases for reconstructing AXI from source artifacts into an approved canonical knowledge system.

## Phases

### Phase 0 — Governance
Establish governance before reconstruction.

Deliverables:
- Reconstruction Charter
- Reconstruction Workflow
- Object Standard
- Lifecycle Standard
- Provenance Policy
- Approval Policy
- Reconstruction Phase Plan
- Definition of Done

### Phase 1 — Repository Inventory
Inventory all source artifacts without interpretation.

### Phase 2 — Classification & Metadata
Assign object types, identifiers, metadata, and lifecycle states.

### Phase 3 — Provenance Mapping
Link every governed object to supporting evidence and sources.

### Phase 4 — Conflict Review
Identify duplicates, superseded material, and unresolved conflicts.

### Phase 5 — Canonical Consolidation
Create candidate canonical objects while preserving history.

### Phase 6 — Founder Review
Approve, reject, or request revisions.

### Phase 7 — Baseline v1.0
Publish the first governed AXI baseline.

### Phase 8 — Development
Only after Baseline v1.0 approval may new development begin.

## Exit Criteria
A phase completes only when its Definition of Done is satisfied and required approvals are recorded.
