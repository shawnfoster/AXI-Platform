# AXI Provenance Policy v1.0

**Document ID:** AXI-PP-001
**Version:** 1.0
**Status:** Founder Review
**Lifecycle:** Draft

## Purpose
Establish mandatory provenance requirements for every governed AXI object to ensure every canonical statement, decision, and artifact is traceable to its source.

## Policy Principles
1. Every governed object shall have provenance.
2. Provenance is preserved permanently.
3. Historical sources are never overwritten.
4. Confidence shall be explicit.
5. Missing provenance shall be recorded, not assumed.

## Required Provenance
Each governed object shall record:
- Object ID
- Source artifact(s)
- Original filename/location
- Source version
- Original author (if known)
- Origin date (if known)
- Reconstruction date
- Reconstruction authority
- Evidence references
- Confidence assessment
- Related decision records
- Supersession status

## Evidence Levels
- Level 1: Primary source
- Level 2: Multiple corroborating sources
- Level 3: Single secondary source
- Level 4: Inference (clearly labeled)
- Level 5: Unknown / unresolved

## Confidence
High, Moderate, Low, or Unknown.

## Rules
- Every canonical claim shall cite supporting evidence.
- Conflicts shall be documented, not hidden.
- Loss of provenance prevents canonization.
- Provenance updates create an audit trail.

## Validation
A governed object cannot enter Approved or Active without complete provenance.

## Definition of Completion
This policy is satisfied when every governed object includes complete provenance and an auditable chain back to its origin.
