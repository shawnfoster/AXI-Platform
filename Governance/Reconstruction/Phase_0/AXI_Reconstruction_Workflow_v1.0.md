# AXI Reconstruction Workflow v1.0

**Document ID:** AXI-RW-001  
**Version:** 1.0  
**Status:** Founder Review  
**Lifecycle:** Draft

## Purpose
Define the governed workflow used to reconstruct AXI into a canonical, evidence-backed knowledge system.

## Workflow

### Phase 1 — Intake
- Register source artifact.
- Assign unique identifier.
- Preserve original without modification.

### Phase 2 — Inventory
- Record filename, source, version, location, checksum (when available), date, owner.

### Phase 3 — Classification
Classify artifact (Publication, Standard, Policy, Workflow, Template, Dataset, Project, etc.).

### Phase 4 — Metadata
Assign required metadata:
- Document ID
- Version
- Status
- Lifecycle
- Provenance
- Dependencies

### Phase 5 — Provenance
Record:
- Original source
- Origin date
- Author (if known)
- Confidence
- Evidence references

### Phase 6 — Conflict Review
Identify duplicates, conflicts, superseded material and unresolved questions.
Record conflicts; do not silently resolve them.

### Phase 7 — Candidate Canon
Prepare reconstructed object for review.
No publication occurs without approval.

### Phase 8 — Founder Review
Founder may:
- Approve
- Reject
- Request Revision
- Defer

### Phase 9 — Canon Publication
Approved artifacts enter the Canon.
Historical artifacts remain preserved.

## Workflow Rules
1. Preserve originals.
2. Never delete history.
3. Never infer without evidence.
4. Record uncertainty.
5. Maintain complete traceability.
6. One approved canonical representation per governed object.

## Inputs
- Source artifacts
- Governance documents
- Reconstruction decisions

## Outputs
- Canonical objects
- Provenance records
- Decision records
- Review logs

## Definition of Completion
The workflow is complete for an artifact only when:
- Inventory complete
- Metadata complete
- Provenance complete
- Conflict review complete
- Founder approval recorded
- Canon or archive destination assigned
