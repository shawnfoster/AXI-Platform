# AXI Definition of Done v1.0

**Document ID:** AXI-DD-001  
**Version:** 1.0  
**Status:** Founder Review  
**Lifecycle:** Draft

## Purpose
Define the measurable completion criteria required before any reconstruction phase, governed object, or publication may be considered complete.

## Governing Principles
1. Completion is determined by evidence, not opinion.
2. Every required criterion shall be satisfied.
3. All approvals shall be recorded.
4. Incomplete work shall not advance to the next phase.

## Definition of Done Criteria

A reconstruction artifact is complete only when all of the following are true:

- Scope completed.
- Required metadata assigned.
- Provenance recorded.
- Lifecycle state assigned.
- Unique identifier validated.
- Dependencies documented.
- Conflicts documented or explicitly deferred.
- Validation rules satisfied.
- Founder review completed.
- Approval status recorded.
- Repository location assigned.
- Version recorded.
- Revision history initialized.
- Audit trail preserved.

## Phase Completion

A reconstruction phase is complete only when:

- Every required deliverable exists.
- Every deliverable has passed its Definition of Done.
- Required approvals are recorded.
- Exit criteria are satisfied.
- Outstanding issues are documented.

## Non-Completion

Work is not complete if any required criterion is missing, even if the content itself is finished.

## Continuous Improvement

This Definition of Done may evolve through governed amendments. Earlier approved versions remain archived.
